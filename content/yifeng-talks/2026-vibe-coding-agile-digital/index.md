很多企业数字化做到今天，状态其实有点尴尬：ERP 上了，流程走了，数据也有了。但一到经营现场，还是有很多“不顺”——临时要个报表得等三天，想对个账得手动导出，跨平台数据拼在一起只能靠 Excel。
这些事不解决，天天难受；真要认真解决，立项、排期、开发，折腾一圈又觉得不值。

这就是很多企业长期缺少的一种能力：**敏捷数字化**——当经营现场出现一个问题时，不走漫长流程，先快速做出一个软件来用上。

这两年技术圈开始流行**Vibe Coding**，我认为能补上企业的这块能力短板。

**Vibe Coding 是什么？**
简单说就是：**你把需求说清楚，AI 帮你把第一版软件快速做出来。**
过去，一个需求要先讲给产品经理，再历经架构、开发、测试，最后才变成一个能跑的东西。现在，这条链路被压缩了——“意图到实现”的距离变短了。

过去一年，我用 Vibe Coding 做了十几个项目：客户需要扫码抽奖，我开发插件集成到官网；客户想每天看到前一天的销售报告，但后台不支持邮件发送，我写了一个自动抓取并发送数据的小工具；每天盯服务器太麻烦，我让服务器主动发邮件汇报状态……
这些项目都不大，但每一个都是真实需求，**每一个都由我一个人从定义问题走到解决问题。** 
这些项目让我确信：一个人，就能具备敏捷数字化能力。

换到企业场景看，有了 Vibe Coding，企业要获得敏捷数字化能力，也许不再需要像过去那样先配齐一整支团队，而是可能先从一种新的角色配置开始。

角色具体怎么配置？我有两点建议。
1. Vibe Coding 不是零门槛就能掌握的，因此你需要找一个逻辑清晰、对技术有热情的计算机专业应届生，成为企业的“敏捷数字化专员”。
2. Vibe Coding 的水平，需要在实战中才能持续提升，因此业务负责人必须把问题和边界讲清楚，让敏捷数字化专员负责实现，把第一版快速推出来，再基于使用反馈，持续迭代。

一段时间之后，你会发现你的企业生长出了敏捷数字化能力。那些曾经让你难受的“不顺”，开始一个个被捋顺。

**数字化不一定非要大张旗鼓地「推倒重来」，很多时候，它就是从解决每一个「顺手不顺手」的小问题开始的。**
